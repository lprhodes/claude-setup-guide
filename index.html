1 Foundations

Start with the right plan

Before anything else: subscribe to Claude Max 5. This gives you access to Claude Code, the most powerful way to work with Claude for software development. Use Opus as your model for everything—it's the most capable for code.

Claude Code

The primary interface. Runs in your terminal or as a VS Code extension. Most power users prefer the terminal directly.

Opus Model

Use Opus for everything. It's the strongest model for reasoning, code generation, and complex multi-step tasks.

💡

2 Terminal Setup

Install the essentials

Open Terminal.app (press Cmd + Space, type "Terminal", hit Enter). Then install these two CLI tools. They'll integrate beautifully with Claude Code later.

GitHub CLI

Lets Claude Code create pull requests, manage issues, and interact with GitHub directly from your terminal.

Terminal 
# Install GitHub CLI
brew install gh

# Log in to your GitHub account
gh auth login

Vercel CLI

The simplest way to deploy personal projects. No AWS complexity, no DevOps overhead—just push and ship.

Terminal 
# Install Vercel CLI
npm install -g vercel

# Log in to your Vercel account
vercel login

Keep Claude Code updated

Anthropic ships major improvements daily. Run this every morning to make sure you're on the latest version.

Terminal 
# Update Claude Code to the latest version
claude install
ⓘ
Run claude install daily rather than relying on auto-update. With the pace of new features landing, you'll want to be on the latest.
3 Tooling & Skills

Extend with skills

Skills.sh adds powerful capabilities to Claude Code. Install it, then add skills from Vercel and Anthropic for the best results.

Terminal 
# Install Skills.sh
npm install -g skills-sh

# Then browse and add skills from trusted publishers
skills install @vercel/ai-sdk
skills install @anthropic/claude-code
⚠
Stick to well-known publishers (Vercel, Anthropic, etc.). Some community skill repositories have hundreds of plugins with high ratings—but third-party skills can contain vulnerabilities. Quality over quantity.
⚠
Security alert: axios npm package. North Korean threat actors have been targeting the axios TypeScript package in the past week. If your projects use axios, update your dependencies immediately.
4 Workflows

Permissions & shell aliases

Claude Code asks for permission before running commands. For beginners, this is a good thing—you'll see exactly what's happening. As you get comfortable, here are the permission tiers:

Default Mode

Recommended for beginners. Claude asks before every file edit and terminal command. Safe and educational.

Auto Mode

Claude makes its own judgment calls on what's safe to run. Currently for Team plan users. Learn more.

Sandbox Mode

Runs commands in an isolated environment. Great for experimenting safely. Sandboxing docs.

Power-user shell aliases

Once you're more comfortable, you can add shortcuts to your shell configuration. These let you launch Claude Code with a single word.

~/.zshrc 
# Continue your last Claude session
claudey() {
    security unlock-keychain ~/Library/Keychains/login.keychain-db
    claude --dangerously-skip-permissions --continue
}

# Start a fresh Claude session
claudey-new() {
    security unlock-keychain ~/Library/Keychains/login.keychain-db
    claude --dangerously-skip-permissions
}
⚠
For beginners: skip the --dangerously-skip-permissions flag. It disables all safety prompts, which means Claude can modify files and run commands without asking you first. Start with default or auto mode until you're confident navigating the terminal.

After editing ~/.zshrc, reload it in any open terminal:

Terminal 
source ~/.zshrc

Remote access

You can access a running Claude Code session from anywhere—your phone or another computer—using the /remote command inside an active session. You can also start a session with the --remote flag.

5 MCP Servers

Supercharge with MCPs

MCP (Model Context Protocol) servers give Claude Code access to external tools and services. Think of them as plugins that let Claude read your browser, manage tasks, search docs, and more.

Documentation

Ref.tools

Fetches relevant documentation context so Claude always references the latest API docs. Well worth paying for.

Project Management

Linear

Fetch tasks, compare against codebases, generate clarifying questions in multi-choice format, then build implementation plans.

AI Components

AI Elements

Vercel's AI UI components. Pair with ai-sdk for building AI-powered interfaces with pre-built, polished components.

Browser Testing

Chrome DevTools

Let Claude interact with your browser—take screenshots, inspect elements, run Lighthouse audits. Swap with Playwright as needed.

Containers

Docker MCP

Read container logs, manage Docker Compose stacks. Ideal for local dev environments with MongoDB, Redis, and more.

AI Development

Vercel AI SDK

The recommended package for all AI-related code. Consistent API across providers, streaming support, and great TypeScript types.

6 Pro Tip

The two-pass UI workflow

Here's a workflow for getting truly polished interfaces: use Claude Code for implementation, then Gemini for visual refinement.

Claude Code builds it → Screenshot the UI → Feed to Gemini → Refined design

The process

1. Build your feature with Claude Code.
2. Take a screenshot of a UI you admire for reference.
3. Open Gemini CLI or AI Studio (not the Gemini website).
4. Give it the screenshot + a detailed prompt covering fonts, colours, padding, margins, and layout.
5. Ask it to generate a design prompt that another AI assistant can execute.
6. Feed that prompt back into Gemini CLI to refine your interface.

💡
This two-pass approach plays to each model's strengths: Claude's deep code reasoning for architecture, and Gemini's visual understanding for UI polish.
